2. O que o mean pool dos tokens textuais produz

O t_pool no PromptGuidedPooler é a média aritmética dos 5 tokens textuais:

h[<s>] + h[<img>] + h[</img>] + h[Analyze] + h[document] ÷ 5 → t_pool
Problema: 3 dos 5 tokens são estruturais (<s>, <img>, </img>). A média é dominada por tokens que não carregam semântica do prompt — o t_pool é mais um artefato de formatação do que uma representação do que foi pedido.

3. Consequência no cross-attention

Essa t_pool corrompida é usada como query para selecionar os patches visuais: 

q = t_pool.unsqueeze(1)           # [B, 1, D] ← média de (struct + semântico)

attn_weights = softmax(q @ K^T / √d)   # K = hidden states dos 1792 patches
v_attended   = attn_weights @ V        # soma ponderada dos patches
Se o t_pool for dominado por tokens estruturais, a query não representa "o que o prompt pede" — representa "como o InternVL3 formata a entrada". A seleção de patches visuais será guiada por um artefato de template, não pela semântica do prompt.

4. Agravante: o prompt é fixo

O EMBEDDING_PROMPT = "<image> Analyze this document" é sempre o mesmo para qualquer imagem, em qualquer batch, em qualquer época de treino.

Isso significa que o t_pool produz o mesmo vetor para todas as imagens — não há variação de prompt que o cross-attention possa aprender a explorar. O pooler é chamado de "prompt-guided" mas na prática o "prompt" é uma constante.

5. O que tornaria o cross-attention genuinamente útil

Opção A — Último token textual

Em LMs causais, o último token da sequência é o que tem mais contexto acumulado — viu todos os tokens anteriores, inclusive toda a imagem. 

# Ao invés de mean:
t_pool = tokens[b, last_text_idx, :]
# = h[▁document] — viu tudo antes

Menos ruído de tokens estruturais. Já é uma prática comum em embedding de LMs causais (ex: E5-Mistral).

Opção B — Prompt variável por tarefa

Usar prompts diferentes por tipo de consulta dá ao cross-attention informação real para discriminar. 

# Exemplos:
"What type of document is this?"
"Extract the invoice values."
"Identify the sender of this letter."

Nesse caso o t_pool varia por amostra → o cross-attention tem sinal real para aprender.

Opção C — Filtrar tokens estruturais

Mean pool apenas sobre tokens com conteúdo semântico, excluindo BOS, tags de imagem e pontuação. 

sem_mask = text_valid & ~is_special_token
t_pool = masked_mean(tokens, sem_mask)

Com prompt fixo, reduz de 5→2 tokens (Analyze + document), menos ruído estrutural.

Opção D — Query aprendida (AttentionPooling)

Voltar para uma query aprendida, mas aplicada apenas sobre os tokens visuais (em vez de toda a sequência). Sem depender do prompt. 

q = self.query.expand(b, -1)     # aprendida
v_attended = CrossAttn(q, visual, visual)

Equivale ao AttentionPooling com máscara visual — sem o problema do t_pool corrompido.

Resumo: Com prompt fixo + mean pooling de 5 tokens (3 estruturais), o PromptGuidedPooler atual se comporta na prática como um cross-attention com query constante — o que é equivalente a um attention pooling sem query aprendida. A opção mais imediata é usar o último token textual como query (Opção A), que não exige mudança de prompt nem filtro de tokens.
