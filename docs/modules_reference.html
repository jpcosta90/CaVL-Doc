Ênfase em como cada pooler trata os tokens visuais e textuais na sequência multimodal.

Contexto: Sequência Multimodal no InternVL3

Antes de entender o que cada pooler faz, é necessário entender o que ele recebe. No InternVL3, a sequência de entrada do LLM é formada pela concatenação de tokens visuais e textuais:

visual
<IMG_CONTEXT> × N
~256 por sub-imagem
⊕
textual
Prompt tokens
~10–30 tokens
→
sequência unificada
LLM hidden states
[B, seq_len, dim]
→
Pooler
→
Head
→
Embedding
[B, 512]
Proporção típica: com MAX_NUM=6, o InternVL3 gera 6 blocos × 256 tokens visuais = 1.536 tokens visuais contra ~20 textuais. Um mean pool ingênuo de toda a sequência produz um embedding ~98% visual — efetivamente ignorando o prompt.
O parâmetro mask de todos os poolers é um tensor booleano [B, seq_len]. Quem controla quais tokens entram no pool é o código chamador — o pooler em si é agnóstico à modalidade. A análise abaixo descreve o comportamento padrão (sem filtro externo).

Poolers — poolers.py

registry: "attention" modality: learned re-weighting

AttentionPooling

Pooler principal do CaVL-Doc. Aprende um vetor de query global (ou múltiplos) que atende a todos os tokens da sequência via Multi-Head Attention. Permite dois modos de operação controlados por num_queries, mantendo retrocompatibilidade de checkpoint.

ParâmetroTipoPadrãoDescrição
hidden_dimint1536Dimensão dos tokens de entrada (InternVL3-2B)
num_headsint8Cabeças de atenção no MHA
num_queriesint11 = modo legado; >1 = multi-query com projeção
dropoutfloat0.0Dropout no MHA

Análise de Modalidade

Tokens Visuais Atuam como keys e values no MHA. Quanto mais a query aprendida apontar para eles, maior o peso. Sem filtro externo, dominam numericamente (>95% da sequência), mas a atenção pode aprender a equilibrar ou a ignorar tokens redundantes.
Tokens Textuais Também atuam como keys e values. São minoria numérica, mas se semanticamente relevantes, a query aprendida pode atribuir-lhes alto peso. Não há distinção estrutural entre modalidades — o balanço é inteiramente aprendido.
Query Aprendida Parâmetro self.query treinado end-to-end. Não distingue modalidades a priori — aprende empiricamente quais tokens (visuais ou textuais) são mais informativos para a tarefa de classificação.
Mask Passada como key_padding_mask no MHA (negada para o padrão PyTorch). Tokens mascarados são excluídos da atenção. Pode ser usada para restringir o pool a uma única modalidade.

Modo Single-Query (num_queries=1)

# query: [1, dim] → [1, B, dim]
q = self.query.unsqueeze(1).expand(1, b, d)
attn_out, _ = self.mha(q, tokens_t, tokens_t)
# out: [1, B, dim] → squeeze → [B, dim]
return self.ln(attn_out.squeeze(0))

Modo Multi-Query (num_queries>1)

# query: [Q, 1, dim] → [Q, B, dim]
q = self.query.expand(-1, b, -1)
attn_out, _ = self.mha(q, tokens_t, tokens_t)
# out: [Q, B, dim] → [B, Q*dim]
flat = attn_out.transpose(0,1).reshape(b, -1)
return self.ln_out(self.proj(flat))
Viés de desequilíbrio: Com toda a sequência como keys/values e sem mask, o gradiente flui predominantemente dos tokens visuais (maioria). Se o objetivo é sensibilidade ao prompt, passar mask ou combinar com balanced pool é recomendado.
registry: "positional_attention" modality: learned re-weighting + ordem

PositionalAttentionPooling

Extensão do AttentionPooling que adiciona encodings posicionais aprendíveis aos tokens antes da atenção. Útil quando a posição relativa entre tokens (ex: ordem de leitura) carrega informação semântica que não está nos hidden states.

ParâmetroTipoPadrãoDescrição
hidden_dimint1536Dimensão dos tokens
num_headsint8Cabeças de atenção
num_queriesint11 = single; >1 = multi-query
max_lenint4096Comprimento máximo para o pos_embed; sequências maiores são interpoladas

Análise de Modalidade

Tokens Visuais Recebem o encoding posicional da sua posição na sequência (índice no input_ids do LLM, não posição espacial no patch grid). O pos_embed é global e sequencial, não espacial — não codifica a grade 2D da imagem.
Tokens Textuais Recebem encoding posicional do índice que ocupam após o bloco visual na sequência. Isso pode beneficiar o pooler ao distinguir, por exemplo, tokens iniciais (BOS, system prompt) de tokens de query.
Interpolação Se seq_len > max_len, o pos_embed é interpolado linearmente — evita OOM mas pode degradar a qualidade posicional para sequências muito longas.
Diferença do base Idêntico ao AttentionPooling em tudo exceto pela adição de tokens = tokens + pe antes do MHA. O tratamento de modalidade é o mesmo (sem distinção estrutural). 
# Diferença-chave em relação ao AttentionPooling:
if seq_len > self.pos_embed.shape[1]:
    pe = F.interpolate(self.pos_embed.transpose(1,2),
                       size=seq_len, mode='linear')
    pe = pe.transpose(1,2)
else:
    pe = self.pos_embed[:, :seq_len, :]

tokens = tokens + pe   # ← única adição em relação ao base
registry: "mean" modality: agnóstico — domínio visual por padrão

MeanPooling

Pooler mais simples: média aritmética sobre os tokens (opcionalmente mascarada), seguida de LayerNorm. Sem parâmetros treináveis além da norm. Serve como baseline e referência para comparação de qualidade de representação.

ParâmetroTipoPadrãoDescrição
hidden_dimint1536Dimensão para o LayerNorm

Análise de Modalidade

Tokens Visuais Sem mask, contribuem proporcionalmente ao seu número — ~98% do peso total para MAX_NUM=6. O embedding resultante é praticamente um resumo visual, insensível ao prompt.
Tokens Textuais Diluídos pela maioria visual. Com mask apontando apenas para tokens textuais (mask = seq != img_ctx_id), o comportamento inverte completamente — funciona como sentence embedding ignorando a imagem.
Mask ponderada Quando mask é fornecida, a média é calculada apenas sobre os tokens True, com divisão pela contagem real (sum_mask.clamp(min=1e-9) — evita divisão por zero).
Caso de uso ideal Baseline ou quando combinado externamente com pooling por modalidade (ex: o _balanced_pool_layer do DocPromptStudy que calcula two separate means e combina 50/50). 
if mask is None:
    pooled = tokens.mean(dim=1)          # media de TODOS os tokens
else:
    mask = mask.unsqueeze(-1).float()
    tokens = tokens * mask
    sum_tokens = tokens.sum(dim=1)
    sum_mask   = mask.sum(dim=1).clamp(min=1e-9)
    pooled     = sum_tokens / sum_mask      # media dos tokens selecionados
return self.ln(pooled)
Para uso multimodal balanceado, este pooler deve ser chamado duas vezes com máscaras distintas (uma para visual, outra para textual) e os resultados combinados externamente. O AttentionPooling faz isso implicitamente via atenção aprendida.
registry: "prompt_guided" modality: cross-modal — texto guia visual requires_visual_mask = True

PromptGuidedPooler

Atenção causal — por que tokens textuais são cross-modal

Sequência: [<s>] [<img>] [v₁···v₁₇₉₂] [</img>] [▁Analyze] [▁document]

Token vₖ → atende a: apenas tokens anteriores (pré-visuais + visuais)  ← cego ao texto
Token tₗ → atende a: TUDO — todos os patches + texto anterior           ← cross-modal

Usa a média dos tokens textuais como query única para cross-attention sobre os patches visuais. O resultado é um resumo visual condicionado no que o texto representa na camada cut_layer. Blend com o t_pool direto via α aprendível iniciando em 0.80 visual / 0.20 texto.

ParâmetroTipoPadrãoDescrição
hidden_dimint1536Dimensão dos tokens
num_headsint8Cabeças no MHA cross-attention
dropoutfloat0.0Dropout no MHA
fixed_alphafloat|NoneNoneSe None, α aprendível; caso contrário fixo

Análise de Modalidade

Tokens Visuais Keys e values no cross-attention. A query textual seleciona os patches mais relevantes. Com prompt fixo ("Analyze this document"), a query é constante — a variação vem do documento.
Tokens Textuais Mean pool dos 5 tokens textuais serve de query e de componente direto no blend. Os tokens pós-visuais (</img>, ▁document) dominam a média por terem representações ricas de documento; os pré-visuais (<s>, <img>) contribuem com ruído mínimo.
α (blend) Inicia em sigmoid(1.386) ≈ 0.80 favorecendo o lado visual. O t_pool já influencia v_attended como query — dar-lhe peso igual direto seria dupla contagem.
Interface Mesma assinatura dos outros poolers: forward(tokens, mask, visual_mask). O CaVLModel computa e passa visual_mask automaticamente quando pooler_type="prompt_guided". 
# t_pool: média dos tokens textuais (cross-modal via atenção causal do LLM)
t_pool = masked_mean(tokens, ~visual_mask)         # [B, D]

# cross-attention: t_pool como query, visual como K/V
v_attended, _ = self.cross_attn(
    query=t_pool.unsqueeze(1),
    key=tokens, value=tokens,
    key_padding_mask=~visual_mask                  # ignora posições não-visuais
)
v_attended = v_attended.squeeze(1)               # [B, D]

a = torch.sigmoid(self._alpha_logit)               # ≈ 0.80 inicial
output = a * self.ln_v(v_attended) + (1-a) * self.ln_t(t_pool)
registry: "modal" modality: assimétrico — visual mean + text attention requires_visual_mask = True novo

ModalPooler (assimétrico por design)

Inverte a lógica do PromptGuidedPooler: aplica mean pooling nos tokens visuais (cujas ativações já codificam distribuição espacial semântica) e atenção aprendida nos tokens textuais (para filtrar os pré-visuais e enfatizar os pós-visuais).

Motivação: heterogeneidade dos tokens textuais

Token      Posição   Atendeu à imagem?   Valor para embedding
─────────────────────────────────────────────────────────────
<s>        0         Não (antes)         ★☆☆☆  ruído
<img>      1         Não (antes)         ★☆☆☆  ruído
v₁···v₁₇₉₂ 2-1793   —  (são visuais)    —
</img>     1794      Sim (todos os patches) ★★★★  rico
▁Analyze   1795      Sim                 ★★★☆  rico
▁this      1796      Sim                 ★★☆☆  médio
▁document  1797      Sim (mais contexto) ★★★★  mais rico

Mean pooling trata todos igualmente. A query aprendida aprende a dar peso ≈0 para <s> e <img> e peso alto para </img> e ▁document — sem supervisão explícita.

ParâmetroTipoPadrãoDescrição
hidden_dimint1536Dimensão dos tokens
num_headsint8Cabeças no MHA de atenção textual
dropoutfloat0.0Dropout no MHA
fixed_alphafloat|NoneNoneSe None, α aprendível iniciando em 0.50

Análise de Modalidade

Tokens Visuais Mean pool uniforme. As ativações dos ~1792 tokens visuais na cut_layer já têm distribuição espacial que funciona como soft-attention — a média captura esse sinal sem parâmetros extras.
Tokens Textuais Query aprendida (text_query) atende sobre os 5 tokens textuais. O MHA com key_padding_mask=~text_valid ignora tokens visuais e padding. Aprende a ponderar </img> e ▁document (pós-visuais, ricos) sobre <s> e <img> (pré-visuais, vazios).
α (blend) Inicia em sigmoid(0) = 0.50 — neutral. Sem o problema de dupla contagem do PromptGuidedPooler: os dois lados são independentes.
Interface Mesma assinatura: forward(tokens, mask, visual_mask). CaVLModel passa visual_mask automaticamente quando pooler_type="modal". 
# visual: mean pool uniforme
vis_w  = visual_mask.unsqueeze(-1).float()
v_pool = (tokens * vis_w).sum(1) / vis_w.sum(1).clamp(min=1e-9)   # [B, D]

# texto: query aprendida atende só nos tokens textuais
text_valid = ~visual_mask
q = self.text_query.expand(B, -1, -1)              # [B, 1, D] — aprendida
t_attended, _ = self.text_attn(
    query=q,
    key=tokens, value=tokens,
    key_padding_mask=~text_valid                    # ignora visuais e padding
)
t_attended = t_attended.squeeze(1)                # [B, D]

a = torch.sigmoid(self._alpha_logit)               # ≈ 0.50 inicial
output = a * self.ln_v(v_pool) + (1-a) * self.ln_t(t_attended)

Comparativo dos Poolers

AttentionPooling

Query aprendida sobre toda a sequência. Sem distinção de modalidade — o balanço visual/texto é inteiramente emergente.

PositionalAttentionPooling

Igual ao anterior + encoding posicional sequencial. Não distingue grade visual de posição textual.

MeanPooling

Média uniforme. Sem parâmetros. Dominado pelos ~1792 tokens visuais a menos que mask restrinja a modalidade.

PromptGuidedPooler

Mean text → query cross-attention sobre visual. Visual side como atenção. Text side como mean pool. α inicia em 0.80 visual.

ModalPooler ★

Mean pool no visual (ativações já funcionam como atenção). Query aprendida no texto (filtra pré-visuais). Design assimétrico por observação empírica.

Heads — heads.py

As heads operam no embedding já poolado [B, hidden_dim] e o projetam para o espaço de embedding final [B, proj_out]. Nenhuma delas tem consciência de modalidade — recebem apenas o vetor do pooler.

registry: "mlp" post-pooling

ProjectionHead (SimCLR style)

Head padrão com dois layers lineares (bottleneck), ativação GELU e normalização L2 opcional. Boa para aprendizado contrastivo puro, mas a projeção pode distorcer a geometria do espaço aprendido pelo pooler.

ParâmetroPadrãoDescrição
input_dim1536Saída do pooler
proj_hidden4096Dimensão do bottleneck intermediário
proj_out512Espaço de embedding final
use_normTrueNormalização L2 na saída (projeção na hiper-esfera)
x = self.ln(x)          # LayerNorm na entrada
x = self.fc1(x)         # [B, 1536] → [B, 4096]
x = self.act(x)         # GELU
x = self.fc2(x)         # [B, 4096] → [B, 512]
if self.use_norm:
    x = F.normalize(x, p=2, dim=-1)   # ||x||₂ = 1
registry: "simple_mlp" legacy

MPProjectionHead (legacy)

Head minimalista: Linear → ReLU → Linear. Sem LayerNorm, sem normalização de saída. Mantida para retrocompatibilidade de checkpoints antigos. 

return self.fc2(self.relu(self.fc1(x)))
# Nenhuma normalização — adequado apenas para loss Euclidiana (ContrastiveLoss)
registry: "residual" recommended for metric learning

ResidualProjectionHead (recomendada para ArcFace/CosFace)

Head com conexão residual (shortcut) para preservar a geometria do espaço do pooler. A normalização L2 na saída é obrigatória para compatibilidade com as losses de margem angular. Usa SiLU (Swish) em vez de ReLU para gradientes mais suaves.

ParâmetroPadrãoDescrição
input_dim1536Saída do pooler
proj_hidden4096Dimensão interna do bloco de transformação
proj_out512Espaço de embedding final
dropout0.1Dropout no bloco principal
normalized_x = self.ln_in(x)
features  = self.net(normalized_x)    # Linear→SiLU→Dropout→Linear
residual  = self.shortcut(normalized_x)  # projeção linear se dim muda
out = features + residual
out = self.ln_out(out)
return F.normalize(out, p=2, dim=-1)  # projeção na hiper-esfera

Connector Layer — layers.py

ViT → LLM bridge

NewConnector

Camada linear simples que faz a ponte entre a saída do ViT (1024) e a dimensão de entrada esperada pelo LLM/head (1536). Substitui o mlp1 original quando há incompatibilidade de dimensões. Opera exclusivamente no domínio visual — recebe features do encoder de visão e as projeta para o espaço do LLM.

ParâmetroPadrãoDescrição
vit_output_dim1024Dimensão de saída do ViT (InternViT-300M)

def forward(self, x: torch.Tensor) -> torch.Tensor:
    # x: [B, n_patches, 1024] → [B, n_patches, 1536]
    return self.linear(x)

Losses — losses.py

Todas as losses operam no espaço de embedding final — recebem vetores [B, proj_out] já processados pela head e são agnósticas a modalidades. Agrupadas por família.

Pairwise Contrastive & Triplet
registry: "contrastive"

ContrastiveLoss (Siamese, distância Euclidiana)

Baseada em pares com label binário: 1 = mesmo documento, 0 = diferentes. Minimiza distância para pares positivos e a empurra além de uma margem para negativos. 

loss_pos = label       * d²
loss_neg = (1-label) * clamp(margin - d, 0)²
# d = ||emb1 - emb2||₂

Hiperparâmetros: margin=1.0. Suporta forward_individual para uso com RL Professor.

registry: "triplet"

OnlineTripletLoss (Batch Hard mining)

Para cada âncora no batch, seleciona automaticamente o positivo mais difícil (maior distância à âncora) e o negativo mais difícil (menor distância à âncora). Requer batches com múltiplas amostras por classe. 

loss = max(0, d(a,p_hardest) - d(a,n_hardest) + margin)

Hiperparâmetros: margin=0.3.

Angular Margin ArcFace, CosFace, ExpFace e variantes Elastic
registry: "arcface"

ArcFaceLoss

Adiciona margem angular m ao ângulo da classe correta: φ = cos(θ + m). Escala logits por s antes do softmax. Requer embeddings normalizados. Padrões: s=64, m=0.50.

registry: "elastic_arcface"

ElasticArcFaceLoss

Relaxa a margem fixa: m ~ N(mean_m, std) amostrado por exemplo no batch. Melhora generalização em dados ruidosos. Padrões: m=0.50, std=0.05.

registry: "cosface"

CosFaceLoss (Large Margin Cosine)

Margem aditiva diretamente no cosseno: φ = cos(θ) - m. Mais estável que ArcFace numericamente. Padrões: s=64, m=0.35.

registry: "elastic_cosface"

ElasticCosFaceLoss

Versão elástica do CosFace: m ~ N(mean_m, std) por elemento do batch. Padrões: m=0.4, std=0.05.

registry: "expface"

ExpFaceLoss (Exponential Angular Margin)

Margem não-linear: T(θ) = cos((θ/π)^m × π). Concentra gradiente em exemplos centrais (ângulo pequeno), ignorando ruidosos. Padrões: s=64, m=0.7.

registry: "elastic_expface"

ElasticExpFaceLoss

Versão elástica do ExpFace: m ~ N(mean_m, std) por elemento. Padrões: m=0.7, std=0.05.

Sub-Center Multi-centróide para variância intra-classe

Substitui o único centróide de classe por K sub-centros. Para cada classe, computa a similaridade com todos os K sub-centros e usa o melhor (max pooling). Ideal para classes com layouts muito distintos (ex: "Invoice" pode ser tabela ou texto corrido).

registry: "subcenter_arcface"

SubCenterArcFaceLoss

K sub-centros com margem angular ArcFace. Pesos: [num_classes × K, proj_out]. Padrões: K=3, s=64, m=0.50. 

cosine_all = F.linear(emb, weight)          # [B, Classes×K]
cosine_all = cosine_all.view(-1, classes, k)  # [B, Classes, K]
cosine, _  = torch.max(cosine_all, dim=2)    # [B, Classes] — melhor sub-centro
registry: "subcenter_cosface"

SubCenterCosFaceLoss

Mesma lógica com margem CosFace. Mais estável em datasets ruidosos. Padrões: K=3, s=64, m=0.35.

registry: "subcenter_circle"

SubCenterCircleLoss

Combina sub-centros (K) com a ponderação dinâmica da Circle Loss. Padrões: K=3, m=0.25, gamma=256.

Circle Loss Ponderação dinâmica de pares

Atribui pesos dinâmicos α a positivos e negativos com base em quão difícil cada par é. Foca em hard negatives e relaxa em pares fáceis. Formula: loss = softplus(logsumexp(logit_neg) + logsumexp(logit_pos)).

registry: "circle"

CircleLoss

Circle Loss padrão. Padrões: m=0.25, gamma=256.

registry: "elastic_circle"

ElasticCircleLoss

Margem elástica: m ~ N(mean_m, std) amostrado independentemente para positivos e negativos. Padrões: m=0.25, std=0.0125, gamma=256.

Isotropic Correção do Cone de Representação

Família projetada para embeddings de alta dimensão de ViT/LLM que sofrem de anisotropia (o "cone effect" — todos os embeddings apontam para uma direção dominante, reduzindo a discriminabilidade). Aplica uma projeção isotrópica antes de computar a similaridade: x_iso = x - (x · u)u, onde u é o eixo do cone estimado por média móvel exponencial dos batches de treinamento.

registry: "iso_arcface"

IsoArcFaceLoss

Projeção isotrópica + ArcFace. Padrões: s=64, m=0.50, momentum=0.9. 

# Atualiza eixo do cone via EMA:
batch_dir = F.normalize(embeddings.mean(dim=0), dim=0)
self.cone_axis = momentum * self.cone_axis + (1-momentum) * batch_dir

# Remove componente do cone:
cone_comp = (embeddings @ self.cone_axis).unsqueeze(1) * self.cone_axis
emb_iso = embeddings - cone_comp   # projeção no complemento ortogonal
registry: "iso_cosface"

IsoCosFaceLoss

Projeção isotrópica + CosFace. Padrões: s=64, m=0.35, momentum=0.9.

registry: "iso_circle"

IsoCircleLoss

Projeção isotrópica + Circle Loss (ponderação dinâmica). Padrões: m=0.25, gamma=256, momentum=0.9.
