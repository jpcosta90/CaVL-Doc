CaVL-Doc — Setup GCP Instance

Guia completo para provisionar uma instância GCP g2-standard-4 (NVIDIA L4) do zero e iniciar o treino ArcDoc.

Especificações da Instância

Máquinag2-standard-4 (4 vCPU, 16 GB RAM)
GPU1× NVIDIA L4 (24 GB)
SODebian 12 Bookworm (debian-cloud/debian-12)
Disco de boot50 GB SSD (NVMe)
Disco de dados200 GB SSD persistente — montado em /mnt/data
ProvisionamentoSPOT (preemptível) — 74% mais barato; checkpoints por época protegem o treino
Zonaus-central1-a (boa disponibilidade de L4)
⚠️ Use sempre 50 GB de boot disk. Com 10 GB o apt fica sem espaço durante a instalação dos drivers NVIDIA.

1 Montar o disco de dados

Verifique se o disco de 200 GB já está montado (GCP às vezes monta automaticamente):

df -h | grep mnt
lsblk

Se /mnt/data não aparecer, formate e monte:

# Substitua sdb pelo nome do disco mostrado no lsblk (pode ser nvme0n2)
sudo mkfs.ext4 /dev/sdb
sudo mkdir -p /mnt/data
sudo mount /dev/sdb /mnt/data
sudo chown -R $USER:$USER /mnt/data

# Persistir montagem no reboot
echo "/dev/sdb /mnt/data ext4 defaults,nofail 0 2" | sudo tee -a /etc/fstab

2 Instalar drivers NVIDIA

# Adicionar repositório non-free do Debian (necessário para nvidia-driver)
sudo tee /etc/apt/sources.list.d/debian-nonfree.list << 'EOF'
deb https://deb.debian.org/debian bookworm main contrib non-free non-free-firmware
deb https://deb.debian.org/debian bookworm-updates main contrib non-free non-free-firmware
deb https://deb.debian.org/debian-security bookworm-security main contrib non-free non-free-firmware
EOF

sudo apt-get update
sudo apt-get install -y nvidia-driver firmware-misc-nonfree linux-headers-$(uname -r)

sudo reboot

Após o reboot, carregue o módulo e verifique:

# O Debian nomeia o módulo como nvidia-current (não nvidia)
sudo modprobe nvidia-current
sudo modprobe nvidia-current-uvm
nvidia-smi

# Carregar módulo automaticamente no boot (evita ter que rodar modprobe manualmente)
echo "nvidia-current" | sudo tee /etc/modules-load.d/nvidia.conf
echo "nvidia-current-uvm" | sudo tee -a /etc/modules-load.d/nvidia.conf
Se o modprobe falhar com "Module not found", o kernel pode ter atualizado durante o apt. Corrija com:

sudo apt-get install -y linux-headers-$(uname -r)
sudo dkms install nvidia-current/535.261.03 -k $(uname -r) --force
sudo modprobe nvidia-current

3 Instalar git e miniconda

sudo apt-get install -y git

# Instalar miniconda no disco de dados (não no boot disk)
curl -fsSL https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh \
    -o /mnt/data/miniconda.sh
bash /mnt/data/miniconda.sh -b -p /mnt/data/miniconda3

echo 'export PATH="/mnt/data/miniconda3/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Aceitar Termos de Serviço do conda
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r

4 Clonar o repositório

cd /mnt/data
git clone https://github.com/jpcosta90/CaVL-Doc.git
cd CaVL-Doc

5 Criar ambiente conda e instalar dependências

conda create -n cavl python=3.11 -y
conda init
source ~/.bashrc
conda activate cavl

# Instalar PyTorch com CUDA 12.1
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# Instalar dependências do projeto
pip install -r requirements.txt

6 Instalar FlashAttention2

FlashAttention2 reduz significativamente o tempo por step nas camadas de atenção do InternVL. Obrigatório para performance adequada.

# Instalar compilador CUDA (necessário para compilar o flash-attn)
conda install -c nvidia/label/cuda-12.1.0 cuda-nvcc -y

# Compilar e instalar flash-attn (~5 min)
pip install flash-attn --no-build-isolation
Se a compilação falhar, verifique se o cuda-nvcc foi instalado corretamente com nvcc --version antes de tentar novamente.

7 Iniciar treino

O script baixa automaticamente o dataset do Hugging Face Hub na primeira execução.

mkdir -p /mnt/data/checkpoints

conda activate cavl
cd /mnt/data/CaVL-Doc

python scripts/training/run_arcdoc_training.py \
  --from-hf \
  --hf-local-dir /mnt/data/final_split3 \
  --run-name ArcDoc_split3_v1 \
  --phase1-num-workers 4 \
  --checkpoint-root /mnt/data/checkpoints \
  --max-steps-per-epoch 250 \
  --gpu-id 0
Retomando após preempção SPOT: adicione --resume ao comando acima. O script retoma automaticamente do último checkpoint salvo em /mnt/data/checkpoints/.

Comandos úteis

Monitorar GPU durante treino

watch -n 2 nvidia-smi

Rodar em background (sobrevive ao fechamento do terminal)

nohup python scripts/training/run_arcdoc_training.py \
  --from-hf \
  --hf-local-dir /mnt/data/final_split3 \
  --run-name ArcDoc_split3_v1 \
  --phase1-num-workers 4 \
  --checkpoint-root /mnt/data/checkpoints \
  --max-steps-per-epoch 250 \
  --gpu-id 0 \
  > /mnt/data/train.log 2>&1 &

tail -f /mnt/data/train.log

Verificar espaço em disco

df -h /
df -h /mnt/data

Expandir boot disk após resize no GCP Console

sudo apt-get install -y cloud-guest-utils
sudo growpart /dev/nvme0n1 1
sudo resize2fs /dev/nvme0n1p1
df -h /
